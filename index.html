Android中的AsyncTask和Thread

發表於 2017-05-28   |   分類於 编程应用   |     |   閱讀次數

最近随着课程的开展终于开始了Android的学习，算是接触移动开发的一个开端吧，和web前端开发类似，移动开发除了前端页面、逻辑层数据层的编写，还有一个不容忽视的关键操作——网络请求。在学习Google与Udacity官方提供的Android教程时，循序渐进的练习引出了UI Thread的概念，以及网络连接必须单独开一个线程以防阻塞主线程的进行，Google教程中给出的为AsyncTask泛型类的实现，而学校课程中老师给的示例代码中用到了Java编程中常用的Thread类加以实现，此处给出两者的分析和区别。

使用示例

AsyncTask\

AsyncTask是一个泛型类，传入的构造参数中Params，Progress和Result分别为:交互的参数、进行中需要与主线程进行通信的数据类型以及线程结束后返回的数据类型。此处为一个发送http请求的功能，中途不需要和UI主线程发生交互，因此参数列表为<URL,Void,String>。

Thread

Java中标准的通过新建Runable类的方式新建一个新的线程。

AsyncTask详解

基本方法

AsyncTask必须在UI主线程中新建和实例化，需要显式调用的方法只有execute()方法，其他都在线程执行过程中由AsyncTask类自行调用，并且调用的方法分为在UI主线程运行的以及子线程运行的，内部通过Handler与UI主线程进行交互，内部则使用ThreadPoolExectutor类进行线程池的调度和管理。对线程的管理和调度更为可控。

主线程中的方法主要有：

  • execute()
  • onProgressUpdate()
  • onPreExecute()
  • onPostExecute()

而后台线程运行的主要方法体在doInBackground()中

源码块分析

可以看到内部初始化局部变量时新建了static的ThreadPoolExecutor以及其他的变量，作用域范围是属于类的，类的属性的作用范围是CLASSPATH，因为一个进程一个VM，所以是AsyncTask控制着进程范围内所有的子类实例。具体的Handler和相关的线程调度在这就不进行讲解了…后续继续…

摩根士丹利面试哇

發表於 2017-05-26   |   分類於 面试整理   |     |   閱讀次數

前段时间偶然间听说摩根士丹利开放了Summer Intern的申请，本着对C++之父Bjarne Stroustrup的敬畏，申请了该项目。这个项目分为两个track，Application Development和Enterprise Engineering，一开始也并不能很好的理解两者的区别，因而官方建议没有明确意愿的可以都选，后期可以再次经过调整。

笔试

摩根士丹利的笔试和国内互联网企业的风格不太一样，使用的是国外的一款在线测评软件，总共有五套题目供选择，Java Development、C++ Development、Cyber Security、Perl/Python Development以及System Engineering，在deadline之前只需完成至少一套就可以了，最终会以最高的一套题目分数为准。比较有意思的是，这款在线测评会根据你的实时正确错误情况，调整题目的难度，题目都是选择题，所以会出现一道很难的题目然后随即就是一道很弱智的读代码题…大概就是上面一道题做错了…
我只选择了Java Development套题进行笔试，题目涵盖JVM的知识以及语言知识，还包括了一些面向对象的概念性和应用型题目，范围与其他笔试并不太大差异

电话面试

从电话面试开始，将都是全英文面试，因而最重要的应该还是能够进行正常有效的交流，能将自己想要表达的表达清楚，面试正式开始时，也应证了我这一点看法，和腾讯电面相比，问题的深度比较浅，比较考验表达能力。根据回忆，大概整理出了以下的问题列表：

  • introduction of yourself, which area are you good at? —— I say front-end development
  • tell something about front-end framework
  • how hash table works, how do you find a value in a hash table, differences between hashtable and hashmap in Java?
  • describe stack & queue
  • describe binary search tree and how it works
  • differences and effects between left join and right join in database
  • describe the utilization of index in database
  • design a systematic solution to subway system using any language you like
  • …

问题主要还是涵盖了最基础的数据结构、数据库以及对系统层面的设计，最后一题会要求你在给出一定的宏观设计的前提下，问到一些细节性的设计，例如某些现实数据你会使用何种数据结构进行组织，你会怎么组织这些模块间的协作，简单描述一下在一个常见的场景下面，你设计的这个系统如何进行协作。总的来看，电话面试作为笔试后的第一次面试，而且是英文面试，还是比较侧重综合基础以及表达能力。

上海终面

最后一场面试是在上海摩根士丹利进行，公司氛围和传统互联网企业的年轻欢快不太相同，比较沉稳严肃。时长共为三小时，一小时的code test，以及两轮技术官的二对一的单独面试。

code test

首先会给出一个form给自己的各方面技术技能打分，后面的面试将会针对这个form以及code test向你提问。code test此次的题目是一个存储和查询公司员工部门的需求，需要

  1. 构建一个数据结构存储表格数据
  2. 构建一个查询该数据结构中数据的函数
  3. 若某字段还有另一树状继承关系，重新修改查询和输出函数

可以使用任意熟悉的语言，手写清晰正确的代码

第一轮单独面试

第一轮主要看了code test的结果，进行了检查和让我解释一下思路，有些bug解决了之后没啥问题了就进行下一个环节了。然后就是对照着自己给技能点的打分提问，这时候就体现出打分要慎重…打高了会让你说自己了解到什么程度，以及有什么相关的经验，比较机智的做法是尽量往自己做过的项目上扯，比较能够说出来东西，纯知识性的回答可能会出现车祸…

第二轮单独面试

第二轮运气比较好，面试官比较和蔼亲和，介绍了两个track的区别并让我选取了一个比较感兴趣的，我选了Application Development，是和金融业务需求开发相关的track，于是接下来面试官开始问我数学问题了…但也不是特别难，主要看反应能力和思维思考的过程吧…

  • 靶子上有一个每秒可以向8个方向移动一格的物体，你瞄准开枪到命中靶子需要两秒（此时物体已经移动了两格），问射哪个格子命中物体的概率最大，是多少？
  • 给出根号10的2位小数表示…嗯…列了一个一元一次方程估算了一下就出来了…

然后这两个问题就把时间花光了…主要是第一个问题扯皮了比较久…互相解释互相扯皮…于是没有进行到将这思路改写成代码的阶段，但看面试官的态度还可以…应该是还ok吧…😂

整场申请面试过程感受下来，还是能体现出一个非纯互联网公司技术岗的风格，各自有各自的侧重点和需求吧。

鹅厂前端面试哇

發表於 2017-05-13   |   分類於 面试整理   |     |   閱讀次數

家在深圳，所以对在鹅厂工作也是一早就憧憬的事情。自己前两天在院里做的大作业和项目基本上也都集中在前端开发，可能自己还是比较喜欢所见即所得的技术方向吧，在鹅厂学长帮助下，也投起了简历准备起了面试。

简历准备

一开始参考网上的简历模板，找了好看的AI模板，保留了项目经历、个人简介、技能点评分以及教育经历等项目，自认为在美观度上还是格式上都比较好，然而询问学长学姐后仍发现许多问题以及需要改进的地方：

  • 项目经历应该以时间倒序排列，最近做的项目排在最前面
  • 和应聘岗位无太大关系的职责和经历需要删除或修改，尽量往目标岗位靠，需展现专业性，而非多样性
  • 项目经历部分着重写自己在项目中起到的不可或缺的作用，强调贡献
  • 技能点评分不直观，最好对自己的语言技能加以文字描述，例如：能熟练使用XXX框架开发XXX……

简历的编写保障了能过简历筛选这一关，准备的过程也是在揣摩HR行为和心理的过程，换位思考也很重要，HR优先想筛选的人，是技术过硬、岗位契合度高、能够直接上岗搬砖的，因此技术岗的简历能够最大程度展现自己的技术能力以及岗位契合度最重要。

面试

内推面试经历了两次，一次电话面，一次视频面。两次面试内容风格不太一致，电话面试主要问到的有数据结构、网络、操作系统、数据库的一些基础知识，以及前端开发上的一些基础知识和了解性知识。

电话面试

因为一开始有整理可能的面试题目，所以大部分还是能够答得出来，但是讲道理应该当时找个地方可以边打电话边看整理资料的，这样也能在功利性上最大程度保证面试质量。被问到的问题涵盖基础知识为主：

  • 排序和搜索的算法(平衡二叉树、B树等)
  • CSS的选择器和伪类
  • 跨域访问的方式
  • web优化的常用手段
  • HTTP请求中缓存的字段？
  • 状态码描述，502,503,400??
  • TCP的握手
  • ping和telnet的区别
  • GET和POST的区别
  • JVM的内存怎么管理的
  • 数据库的性能优化？？
  • 常见的网络攻击有哪些，怎么预防
  • ES6的一些特性
  • 有没有关注一些开源的项目，以及更新的动向
  • Node.js 的架构有没有了解
  • 能不能说几种设计模式
  • session在服务器端是怎么识别客户端机的
  • ……

视频面试

面试我的是一个MIG的技术主管，比较和善，相比于电话面试，视频面试还是比较紧张的，面试形式也较为随意，没有电话面试那么程式化，主要询问了一些关于做过的项目的内容，后半部分又问回了基础知识，后来问他的时候他说问基础知识是看看再学校有没有认真学习，大概……在校生的那点技术经验比较水吧，还是看态度和进取心和学习能力为主，大概的问题列表如下：

  • 排序算法
  • 搜索算法
  • 网络的七层模型
  • 操作系统调度
  • 说一个最能体现技术的项目，最困难的地方是什么
  • 一个字符串，统计各字符的数目的算法有哪些
  • TCP协议的三次握手以及传输控制的内容
  • ……

反思

说实话，面试两场下来发现应聘也就是考察基础知识的掌握程度和学习能力，以及在该岗位领域的热情和对新技术的了解。还是需要加强基础知识的巩固和完善，多刷题，多背诵😂
