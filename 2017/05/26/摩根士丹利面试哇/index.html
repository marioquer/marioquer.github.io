摩根士丹利面试哇

發表於 2017-05-26   |   分類於 面试整理   |     |   閱讀次數

前段时间偶然间听说摩根士丹利开放了Summer Intern的申请，本着对C++之父Bjarne Stroustrup的敬畏，申请了该项目。这个项目分为两个track，Application Development和Enterprise Engineering，一开始也并不能很好的理解两者的区别，因而官方建议没有明确意愿的可以都选，后期可以再次经过调整。

笔试

摩根士丹利的笔试和国内互联网企业的风格不太一样，使用的是国外的一款在线测评软件，总共有五套题目供选择，Java Development、C++ Development、Cyber Security、Perl/Python Development以及System Engineering，在deadline之前只需完成至少一套就可以了，最终会以最高的一套题目分数为准。比较有意思的是，这款在线测评会根据你的实时正确错误情况，调整题目的难度，题目都是选择题，所以会出现一道很难的题目然后随即就是一道很弱智的读代码题…大概就是上面一道题做错了…
我只选择了Java Development套题进行笔试，题目涵盖JVM的知识以及语言知识，还包括了一些面向对象的概念性和应用型题目，范围与其他笔试并不太大差异

电话面试

从电话面试开始，将都是全英文面试，因而最重要的应该还是能够进行正常有效的交流，能将自己想要表达的表达清楚，面试正式开始时，也应证了我这一点看法，和腾讯电面相比，问题的深度比较浅，比较考验表达能力。根据回忆，大概整理出了以下的问题列表：

  • introduction of yourself, which area are you good at? —— I say front-end development
  • tell something about front-end framework
  • how hash table works, how do you find a value in a hash table, differences between hashtable and hashmap in Java?
  • describe stack & queue
  • describe binary search tree and how it works
  • differences and effects between left join and right join in database
  • describe the utilization of index in database
  • design a systematic solution to subway system using any language you like
  • …

问题主要还是涵盖了最基础的数据结构、数据库以及对系统层面的设计，最后一题会要求你在给出一定的宏观设计的前提下，问到一些细节性的设计，例如某些现实数据你会使用何种数据结构进行组织，你会怎么组织这些模块间的协作，简单描述一下在一个常见的场景下面，你设计的这个系统如何进行协作。总的来看，电话面试作为笔试后的第一次面试，而且是英文面试，还是比较侧重综合基础以及表达能力。

上海终面

最后一场面试是在上海摩根士丹利进行，公司氛围和传统互联网企业的年轻欢快不太相同，比较沉稳严肃。时长共为三小时，一小时的code test，以及两轮技术官的二对一的单独面试。

code test

首先会给出一个form给自己的各方面技术技能打分，后面的面试将会针对这个form以及code test向你提问。code test此次的题目是一个存储和查询公司员工部门的需求，需要

  1. 构建一个数据结构存储表格数据
  2. 构建一个查询该数据结构中数据的函数
  3. 若某字段还有另一树状继承关系，重新修改查询和输出函数

可以使用任意熟悉的语言，手写清晰正确的代码

第一轮单独面试

第一轮主要看了code test的结果，进行了检查和让我解释一下思路，有些bug解决了之后没啥问题了就进行下一个环节了。然后就是对照着自己给技能点的打分提问，这时候就体现出打分要慎重…打高了会让你说自己了解到什么程度，以及有什么相关的经验，比较机智的做法是尽量往自己做过的项目上扯，比较能够说出来东西，纯知识性的回答可能会出现车祸…

第二轮单独面试

第二轮运气比较好，面试官比较和蔼亲和，介绍了两个track的区别并让我选取了一个比较感兴趣的，我选了Application Development，是和金融业务需求开发相关的track，于是接下来面试官开始问我数学问题了…但也不是特别难，主要看反应能力和思维思考的过程吧…

  • 靶子上有一个每秒可以向8个方向移动一格的物体，你瞄准开枪到命中靶子需要两秒（此时物体已经移动了两格），问射哪个格子命中物体的概率最大，是多少？
  • 给出根号10的2位小数表示…嗯…列了一个一元一次方程估算了一下就出来了…

然后这两个问题就把时间花光了…主要是第一个问题扯皮了比较久…互相解释互相扯皮…于是没有进行到将这思路改写成代码的阶段，但看面试官的态度还可以…应该是还ok吧…😂

整场申请面试过程感受下来，还是能体现出一个非纯互联网公司技术岗的风格，各自有各自的侧重点和需求吧。
